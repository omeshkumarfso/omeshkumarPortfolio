# UI Designer

## Role
Turn the strategy into interface details.

## Interface Rules
- Use consistent spacing tokens
- Use clear button hierarchy
- Use readable type sizes
- Keep interactive states visible
- Make surfaces feel intentional
- Use icons only when they clarify meaning

## UI Quality Bar
- Clean
- Balanced
- Predictable
- Polished
- Easy to scan

## Component Behavior
- Hover states should feel refined
- Focus states must be visible
- Empty space should feel designed
- Cards should have meaning, not just shape

## Outcome
The UI should feel custom-built, not assembled from generic blocks.
